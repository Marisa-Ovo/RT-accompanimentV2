# 系统架构说明

## 模块职责

| 模块 | 职责 | 不做什么 |
|------|------|----------|
| `my_tokenizer.py` | 编解码：pianoroll ↔ token，构建训练序列/生成schedule | 不做 I/O、不做模型推理 |
| `model.py` | 纯模型推理：接收 tensor + schedule → 自回归生成 | 不做文件加载、不调 tokenizer |
| `Token2Midi.py` | beats 解码 → MIDI 文件输出 | 不做生成、不知道 model 返回格式 |
| `inference.py` | 编排层：加载数据 → tokenizer → model → converter | 唯一涉及全链路的胶水代码 |
| `PianoDataset.py` | 训练数据加载 + 增强 + batching | 不做推理 |
| `config.py` | 所有超参数和 token ID 定义 | — |

## 训练流水线

### 数据流

```
  npz files
      │
      ▼
 ┌─────────────────────────────┐
 │ PianoDataset.__getitem__()  │
 │   ├─ np.load(npz)           │  I/O
 │   ├─ pitch_shift 数据增强    │  增强
 │   └─ tokenizer.             │
 │       build_training_sequence│  编码
 │   → (input_ids, labels)     │
 └──────────┬──────────────────┘
            │ List[{input_ids, labels}]
            ▼
 ┌─────────────────────────────┐
 │ DataCollator + DataLoader   │
 │   ├─ dynamic padding        │
 │   ├─ attention_mask         │
 │   └─ BucketBatchSampler     │  按长度分桶减少 padding
 │   → {input_ids, labels,     │
 │      attention_mask} batch   │
 └──────────┬──────────────────┘
            │
            ▼
 ┌─────────────────────────────┐
 │ TransformerTrainer.train()  │
 │   ├─ Accelerator (DDP/fp16) │
 │   ├─ model.forward(batch)   │  只调 forward，不调 generate
 │   ├─ backward + optimizer   │
 │   ├─ periodic eval          │
 │   └─ save checkpoint        │
 └─────────────────────────────┘
```

### 模块职责（训练）

| 模块 | 职责 | 不做什么 |
|------|------|----------|
| `train.py` | 训练入口：创建 dataset/dataloader/model/trainer，启动训练 | 不含训练循环逻辑 |
| `trainer.py` | 训练循环：optimizer、scheduler、DDP、eval、checkpoint | 不做数据加载、不做 tokenization |
| `PianoDataset.py` | 文件 I/O + 增强 + 截断，委托 tokenizer 编码 | 不含编解码实现 |
| `my_tokenizer.py` | `build_training_sequence()` → (input_ids, labels) | 不做 I/O、不做增强 |
| `config.py` | `TrainingConfig` + `ModelConfig`，所有超参数的唯一定义处 | — |

### 接口定义（训练）

```python
# PianoDataset.__getitem__(idx)
→ {'input_ids': torch.Tensor, 'labels': torch.Tensor}

# DataCollatorForVariableLengthLM(features)
→ {'input_ids': (B, L), 'labels': (B, L), 'attention_mask': (B, L)}

# TransformerTrainer(config, model, train_dataloader, test_dataloader)
trainer.train()  # 完整训练流程

# train.py::main()
# create_datasets → create_dataloaders → initialize_model → TransformerTrainer → train
```

### 关键设计

- **tokenizer 只被 Dataset 调用** — trainer/train.py 完全不知道 token 格式
- **trainer 只看 batch dict** — `{input_ids, labels, attention_mask}`，与音乐领域解耦
- **train.py 是工厂** — 负责组装所有组件，但不含任何循环逻辑
- **Selective Loss 在 tokenizer 中完成** — labels 中 mel/bar/beat 位置已填 `pad_token_id`，trainer 无需额外 mask

---

## 设计原则

1. **model 不碰磁盘** — 输入是 tensor，输出是 tensor/list，无 `os`/`np.load`
2. **converter 不知道 model** — 接口是原始类型 `(mel_beats, acc_beats, tempo)`，不接受 model 返回的 dict
3. **inference 是唯一胶水** — 文件 I/O、tokenizer 调度、结果组装全在这里
4. **tokenizer 是编解码核心** — 通过构造注入共享，不重复创建实例
